1. WHY USE UI FRAMEWORKS LIKE VUE!?

    : 자바스크립트에서," DOM 에 접근하기 위해", "하나의 요소에 접근하려고 할 때마다",
        "하나씩 지정해야하는 코드" ,"무언가를 생성", "수신기를 다는 행위", "데이터에 접근"

        전부 하나의 일에 하나의 동작을 수행하므로, 약간의 기능을 위해 수많은 코드를 적어야한다.

        WE WRITE IMPERATIVE CODE ( 피할 수 없는 단계별 지침에 따른 코드를 작성해야한다) 선택하고 함수쓰고 수신달고 ...

        사용할 코드를 쓰는 상황에서는 좋겠지만, 

        "인터페이스 코드" 를 작성하는 상황에서는 "그렇지 않다!", 오히려 HTML 처럼 편하게 인터페이스의 위치만 보면서 작성하면된다.

        즉, 최종구조를 알 수 있다는 것이다.

        [VUE 같은 UI FRAMEWORKS 는]

        "자바스크립트에서" 작성되는 "인터페이스코드 (li 만들고 append 하고...)"를 간단하게 만들 수 있다!

2. 어떤 FRAMEWORKS 를 사용해야할까?

    1.  REACT       : 가장 유명, 많은 자바스크립트 지식 필요
    2.  VUE         : 적당히 유명, 적당한 자바크립트 지식 필요
    3.  ANGURLAR    : 적당히 유명, 많은 자바스크립트 지식, 특히, 타입스크립트 사용해야함.

    ==> VUE로 배우자!

    나중에 REACT 배우다보면, 자바스크립트 지식도 뿜뿜
